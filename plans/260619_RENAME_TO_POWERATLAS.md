## Intent

### Problem statement & desired outcomes

The project is currently named "kiro-orchestrator" / "Kiro Orchestrator" across all surfaces (Python package, CLI command, UI, config paths, Windows artifacts). It needs to be rebranded to "PowerAtlas" and published to a new public GitHub repository at https://github.com/QuentinSylvestre/PowerAtlas.

### Success criteria

1. Python package renamed to `power_atlas`, CLI command to `power-atlas`, all source imports updated
2. All user-facing strings show "PowerAtlas" (UI title, tray tooltip, console messages, README)
3. Config directory moved to `%LOCALAPPDATA%\power-atlas\`; one-time migration copies existing config from old path
4. Autostart shortcut cleaned up (old `.lnk` removed, new one created if autostart was enabled)
5. Windows mutex renamed to `PowerAtlasMutex`
6. Tray icon letter changed from "K" to "P"
7. Git remote added and full history pushed to `https://github.com/QuentinSylvestre/PowerAtlas`
8. All tests pass after rename
9. `memory/MEMORY.md` and `plans/ROADMAP.md` updated; archived plan files untouched

### Scope boundaries & non-goals

- **IN**: all mechanical renames, one-time migration (disposable), git remote setup + push, workspace folder rename
- **OUT**: proper icon/branding (later), Linux support, renaming archived `plans/done/` files, rewriting git history
- **DO NOT RENAME**: `data.py` references to `.kiro`/`Kiro-Cli` paths and `launcher.py` references to `kiro-cli` command — these are external tool paths
- **SEQUENCING CONSTRAINT**: the workspace folder (`kiro-orchestrator` → `PowerAtlas`) must be renamed. This invalidates the cwd of any running process. All code changes + commit must complete before the folder rename. The remote add + push happens from the new folder path. The venv must also be recreated after the folder move (absolute paths inside it become stale).

## 3) Design Decisions

| Decision | Choice | Alternatives considered | Rationale |
|---|---|---|---|
| Python package name | `power_atlas` | `poweratlas` | PEP 8 convention for multi-word package names; more readable |
| CLI command name | `power-atlas` | `poweratlas` | Kebab-case matches pyproject convention and is standard for CLI tools |
| Config dir name | `power-atlas` | `PowerAtlas`, `power_atlas` | Matches CLI command; lowercase convention for appdata dirs |
| Config migration | One-time disposable code at startup | Permanent migration feature, no migration | One user, settings recreatable but migration is cheap; disposable avoids permanent dead code |
| Git history | Push as-is | Squash to single commit | Preserves development trail; old names in history are harmless |
| Archived plan files | Leave untouched | Rename references | Historical record; not active code |
| Autostart shortcut | Delete old + recreate in migration | Manual cleanup | Automated is reliable; same disposable philosophy |
| Tray icon | Change "K" to "P" | Proper icon now | Deferred to later; minimal change for now |
| Folder rename | Last step after all code changes committed | First step | Code changes need to happen in a valid cwd; folder rename invalidates running sessions |

### Phase 3: One-time config migration [QA]

**Goal**: Add disposable startup code that migrates config from old path to new path and cleans up the old autostart shortcut.

**File scope**: `src/power_atlas/__main__.py`

**Changes**:

Add a `_migrate_legacy()` function called once at the start of `_run_foreground()`, **before** any `CONFIG_DIR.mkdir()` call or `load_config()`:

```python
def _migrate_legacy() -> None:
    """One-time migration from kiro-orchestrator to power-atlas. Remove after first run."""
    import shutil
    localappdata = os.environ.get("LOCALAPPDATA") or str(Path.home() / "AppData" / "Local")
    old_config = Path(localappdata) / "kiro-orchestrator"
    new_config = Path(localappdata) / "power-atlas"
    # Migrate config dir (skip if new already exists or old doesn't)
    if old_config.exists() and not new_config.exists():
        try:
            shutil.copytree(old_config, new_config)
        except OSError:
            # Partial copy — remove so next launch retries
            shutil.rmtree(new_config, ignore_errors=True)
            return
        print(f"Migrated settings from {old_config} to {new_config}")
    # Clean up old autostart shortcut
    appdata = os.environ.get("APPDATA", "")
    old_shortcut = Path(appdata) / "Microsoft" / "Windows" / "Start Menu" / "Programs" / "Startup" / "Kiro Orchestrator.lnk"
    if old_shortcut.exists():
        old_shortcut.unlink()
        # Re-create autostart with new name
        try:
            from .autostart import enable
            enable()
        except Exception:
            print("Warning: could not re-create autostart shortcut after migration")
```

**Exit criteria**:
- [x] Running with old config at `%LOCALAPPDATA%\kiro-orchestrator\` copies it to `%LOCALAPPDATA%\power-atlas\`
- [x] Old `"Kiro Orchestrator.lnk"` is removed if present
- [x] New `"PowerAtlas.lnk"` is created if old shortcut existed
- [x] Does not error if old paths don't exist (fresh install scenario)
